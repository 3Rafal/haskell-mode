13 Interactive Haskell

interactive-haskell-mode is the minor mode that provides interactive editing features. Most of these features are provided by querying the haskell REPL processes.

13.1 Enable this mode

Run the function M-x interactive-haskell-mode

If you want to enable this mode permanently, then hook this mode to haskell-mode by putting this line in your .emacs file: 

(add-hook 'haskell-mode-hook 'interactive-haskell-mode)

13.3 More on M-.

The M-. functionality works if the file in the buffer is loaded into the REPL (C-c C-l). The file in the current buffer can only be loaded if the code doesn’t contain errors. When we can succesfully load the file in the current buffer into REPL, we can get accurate go-to function definition with M-.

13.3.1 M-. with hasktags

Sometimes we might want to use the M-. functionality even when the code doesn’t compile. This is when we use TAGS.

First install hasktags.

You can run either 

$ cabal install hasktags

or 

$ stack install hasktags

Then we run M-x haskell-mode-generate-tags to generate tags. Now we can use M-. functionality even when the code doesn’t compile.

But running M-x haskell-mode-generate-tags everytime can be a too much typing to do just a M-.. We set haskell-tags-on-save to t and the TAGS will be generated on saving the file in buffer.
